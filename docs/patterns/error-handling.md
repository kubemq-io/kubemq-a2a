# Error Handling

A2A errors fall into two categories: **transport errors** (KubeMQ cannot reach or proxy the agent) and **application errors** (the agent itself returns an error).

## Transport vs Application Errors

| Category | Source | Examples |
|----------|--------|----------|
| Transport | KubeMQ gateway | Agent not found, timeout, connection refused, concurrency limit |
| Application | Agent server | Agent returns HTTP 500, returns JSON-RPC error |

Both are returned as JSON-RPC 2.0 error responses, but transport errors use KubeMQ-specific error codes while application errors reflect the agent's response.

## JSON-RPC Error Response Format

```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "error": {
    "code": -32002,
    "message": "agent not found: nonexistent-agent"
  }
}
```

## KubeMQ-Specific Error Codes

### -32001: Agent Timeout

The agent did not respond within the timeout period.

```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "error": {
    "code": -32001,
    "message": "agent timeout: echo-agent-01"
  }
}
```

**Triggered by:** Agent does not respond within `params.configuration.timeout` seconds (capped at MaxTimeoutSeconds=3600, plus ~10s server-side gateway buffer).

### -32002: Agent Not Found

No agent is registered with the given `agent_id`.

```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "error": {
    "code": -32002,
    "message": "agent not found: nonexistent-agent"
  }
}
```

**Triggered by:** The `agent_id` in the URL path is not present in the registry.

### -32603: Internal Error

Concurrency limit exceeded, response too large, or agent returned HTTP 500.

```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "error": {
    "code": -32603,
    "message": "internal error: concurrency limit exceeded"
  }
}
```

**Triggered by:**
- Per-agent concurrency limit (100) exceeded
- Agent response body exceeds 10 MB
- Agent returns HTTP 500

## Standard JSON-RPC Error Codes

### -32700: Parse Error

```json
{
  "jsonrpc": "2.0",
  "id": null,
  "error": {
    "code": -32700,
    "message": "parse error: invalid JSON"
  }
}
```

**Triggered by:** Invalid JSON body (e.g., `{invalid json!!!}`) or wrong `Content-Type` header (e.g., `text/plain` instead of `application/json`).

### -32600: Invalid Request

```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "error": {
    "code": -32600,
    "message": "invalid request: missing method field"
  }
}
```

**Triggered by:**
- Missing `method` field
- Invalid `jsonrpc` version (e.g., `"1.0"` instead of `"2.0"`)
- Empty `agent_id`
- Invalid `agent_id` format (uppercase, special characters)

### -32601: Method Not Found

Reserved by JSON-RPC 2.0. KubeMQ currently forwards all methods to agents, so this code is not generated by the gateway.

### -32602: Invalid Params

Reserved by JSON-RPC 2.0 for malformed parameter objects.

## HTTP Status Code Mapping

When an agent returns an HTTP error status, KubeMQ translates it to a JSON-RPC error:

| Agent HTTP Status | JSON-RPC Error | Message Contains |
|-------------------|---------------|-----------------|
| 500 | `-32603` | "500" or error details |
| 502 | `-32603` | "unavailable" or "502" |
| 503 | `-32603` | "unavailable" or "503" |
| 504 | `-32001` | "timeout" or "504" |

## Connection Refused

When an agent's registered URL is unreachable (server not running, port not listening):

```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "error": {
    "code": -32603,
    "message": "internal error: connection refused"
  }
}
```

The error message typically contains one of: "unreachable", "refused", or "unavailable".
